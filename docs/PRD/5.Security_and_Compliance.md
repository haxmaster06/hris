# SECURITY PRINCIPLES

## Zero Trust Model

Tidak ada request yang dipercaya secara default.

Setiap request wajib:

* Authenticated
* Authorized
* Validated
* Logged

---

## Least Privilege Access

User hanya bisa akses data yang benar-benar diperlukan.

Contoh:

* Staff hanya bisa lihat data sendiri
* Manager hanya bisa lihat team
* HR full access terbatas scope tenant

---

## Defense in Depth

Security berlapis:

* Network
* Application
* Database
* Storage
* API Gateway

# AUTHENTICATION SECURITY

## JWT Strategy

* Access Token: 15–60 menit
* Refresh Token: 7–30 hari
* Rotation: Required

---

## MFA (Multi Factor Authentication)

Supported methods:

* OTP (Email / SMS)

* SSO (Future)

## Password Policy

Minimum:

* 12 characters
* Uppercase
* Lowercase
* Number
* Symbol

* Password history (5 terakhir)
* Block common passwords
* Forced reset after compromise

---

## Session Security

* Device tracking
* IP tracking
* Session invalidation
* Concurrent session limit (configurable)

# AUTHORIZATION (RBAC + ABAC)

## RBAC (Role Based Access Control)

Entities:

* Role
* Permission
* User Role Mapping

---

## Permission Format

```text
module.action.scope
```

Contoh:

```text
employee.read.own
employee.read.team
employee.read.all
payroll.read.own
payroll.read.all
```

---

## ABAC (Attribute Based Access Control)

Rules berdasarkan atribut:

* company_id
* department_id
* position_level
* employment_type

Contoh:

Manager hanya bisa akses employee dengan:

```text
employee.department_id = user.department_id
```

---

# DATA ACCESS CONTROL

## Tenant Isolation

Semua query wajib:

```sql
WHERE tenant_id = ?
```

---

## Row Level Security (PostgreSQL)

Recommended:

* Enable RLS per table
* Enforce tenant isolation at DB level

---

## Field Level Security

Sensitive fields:

* salary
* tax_id
* bank_account
* medical info

Contoh rule:

* HR only
* Payroll only
* Finance only

---

# API SECURITY

## Rate Limiting

Global API:

* 100 req/min/user

Auth endpoint:

* 5 req/min/IP

Payroll endpoint:

* 10 req/min/user

---

## API Protection

* API Key (internal services)
* JWT Validation
* Signature verification (optional for integrations)

---

## Input Validation

Mandatory:

* Schema validation
* Type validation
* Sanitization
* Whitelist fields only

---

## SQL Injection Protection

* ORM only (no raw query default)
* Query builder strict mode
* Prepared statements

---

## XSS Protection

* Output encoding
* CSP headers
* Sanitized rich text input

---

## CSRF Protection

* Token-based protection (for web session mode)
* SameSite cookie enforcement

---
