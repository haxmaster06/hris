# PRD PART 6

# PRODUCT ROADMAP & MICROSERVICES MIGRATION BLUEPRINT

---

# PRODUCT ROADMAP OVERVIEW

HRIS ini dibangun dengan strategi:

> Modular Monolith First → Event Driven → Microservices Later

Tujuan:

* Cepat rilis
* Tidak over-engineering di awal
* Tetap siap scale enterprise

---

* Organization Structure
* Attendance Basic

# MIGRATION STRATEGY (STEP BY STEP)

---

## STEP 1 — STRANGLE PATTERN

* Keep monolith alive
* Extract 1 service at a time
* Route via API Gateway

Example:

```text id="mig001"
Monolith HRIS
   ↓
Payroll Service (Extracted)
   ↓
API Gateway routes payroll requests
```

---

## STEP 2 — EVENT DRIVEN ENABLEMENT

Introduce event flow:

EmployeeCreated
AttendanceLogged
PayrollRequested

All events pushed to:

* domain_events table
* message queue

---

## STEP 3 — DATABASE SPLIT (OPTIONAL LATER)

Phase:

### Phase A (Shared DB)

All modules share PostgreSQL

---

### Phase B (Schema Split)

Each service has schema:

* employee_schema
* payroll_schema

---

### Phase C (Database Split)

Each service has DB:

* employee_db
* payroll_db

---

# EVENT DRIVEN ARCHITECTURE

---

## EVENT BUS

Options:

* Redis Streams (MVP)
* RabbitMQ (Recommended)
* Kafka (Enterprise scale)

---

## EVENT FLOW

```text id="evt001"
EmployeeCreated
   ↓
Payroll Service listens
   ↓
Attendance Service updates
   ↓
Notification Service triggers
```

---

## EVENT RULES

* Event is immutable
* Event must be idempotent
* Event must have versioning

---

# CQRS READ MODEL STRATEGY

---

## WRITE MODEL

* Transactional database
* Strict validation

---

## READ MODEL

* Optimized tables
* Denormalized data
* Fast query layer

---

Example:

```text id="cqrs01"
employee_master (WRITE)

employee_dashboard_view (READ)
```

# MIGRATION RISKS

---

## 1. DATA INCONSISTENCY

Mitigation:

* Event sourcing
* Outbox pattern

---

## 2. DISTRIBUTED COMPLEXITY

Mitigation:

* Start monolith modular
* Extract slowly

---

## 3. DEBUG DIFFICULTY

Mitigation:

* Distributed tracing
* Central logging

---

## 4. NETWORK LATENCY

Mitigation:

* Reduce sync calls
* Use async events
